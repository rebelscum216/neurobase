# Working notes

Scratch space for **thinking in progress** — investigation logs, spike write-ups,
half-formed designs, "here's what I found poking at Codex rollouts today." Unlike
[ADRs](../adr/README.md) and the [spec appendix](../neurobase-spec-appendix.md),
notes are **not contracts** and carry no authority. They're the lab notebook.

## Conventions

- One file per topic or session: `YYYY-MM-DD-short-slug.md`
  (e.g. `2026-07-08-codex-rollout-probe.md`).
- Start each note with a date and a one-line "what this is."
- When a note produces a real decision, promote the conclusion to an
  [ADR](../adr/README.md) and link back to the note. The note stays as the evidence
  trail.
- Absolute dates only. Sanitize anything captured from real sessions (no secrets,
  no personal paths that matter).

## Layout

| Path | Purpose |
|---|---|
| `spikes/` | Raw write-ups for the S1–S6 de-risking spikes, feeding the ADRs. See [spikes/](spikes/). |
| `*.md` (here) | Everything else — general investigation and scratch. |

_This directory is the designated place to "drop working notes" — no need to ask
where they go._
