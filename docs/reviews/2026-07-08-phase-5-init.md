# Review: Phase 5-init — Codex hook installer

## Brief  _(Author — Claude)_

**Intent.** Finish the Phase 5 installer follow-up for Codex: `neurobase init
--agent codex` now writes Codex hook configuration with the same
consent-first, idempotent, backed-up discipline as the Claude installer, while
respecting Codex's distinct discovery/trust rules from spec §7 and ADR-0006.
This closes the code/test side of the Codex installer that was deliberately
deferred from the Phase 5 core adapter review.

**Scope.** Branch `phase-5-init`, `git diff --cached`. Key files:
- `src/neurobase/adapters/codex/install.py` — new Codex installer: reads/writes
  `hooks.json`, builds CamelCase `SessionStart`/`Stop` command-string handlers,
  detects Neurobase-owned `hook codex` entries with a path-anchored fence, and
  surgically merges a project table into `~/.codex/config.toml`.
- `src/neurobase/cli/__init__.py` — `init --agent codex` dispatch, consent
  diff rendering for one or two target files, backup integration, write calls,
  and the user-facing Codex trust-gate reminder.
- `tests/test_codex_install.py` — installer unit tests for hook ownership,
  idempotence, malformed-file refusal, path scopes, and surgical TOML merge.
- `tests/test_cli_init.py` — CLI integration tests for project/user Codex init,
  consent abort, idempotence, backups, malformed inputs, and preserved config.
- `docs/adr/0006-codex-hook-command-tokenization-and-trust.md` +
  `docs/adr/README.md` — records the live spike that confirmed Codex tokenizes
  command strings, sends hook payloads on stdin, and re-prompts trust after
  hook edits.
- `AGENTS.md`, `docs/README.md`, `docs/code-review-relay.md`, and
  `.claude/skills/xcode-review/SKILL.md` — update the project map and replace
  the old `code-review-relay`/`codexprompt` Claude skills with the consolidated
  `xcode-review` Author-role skill.
- `src/neurobase/adapters/codex/__init__.py` — updates the package map now that
  the installer is live.

**Focus areas.**
- **Codex config merge safety.** `merge_config()` edits TOML text surgically to
  preserve comments and unrelated tables. Does it correctly find quoted
  `[projects."<repo>"]` tables, avoid corrupting forms it cannot rewrite, and
  remain idempotent?
- **Ownership fences.** `_OWNED_RE` should replace only Neurobase-owned
  `neurobase hook codex ...` handlers, never prose mentions, unrelated tools,
  or Claude `hook claude` handlers.
- **Installer behavior.** Project scope must write both `<repo>/.codex/hooks.json`
  and the global `~/.codex/config.toml` project table; user scope should write
  only `~/.codex/hooks.json`. Both paths must show exact diffs, ask consent
  unless `--yes`, back up pre-existing files, write atomically, and be
  idempotent.
- **Spec §7 fidelity.** Hooks are CamelCase (`SessionStart`, `Stop`), handlers
  are `type:"command"` with absolute shim command strings, project scope sets
  `hooks = ".codex/hooks.json"` and `trust_level = "trusted"`, and the CLI tells
  the user Codex must approve the edited hook before it fires.
- **Skill rename cleanup.** The deletion of the old `code-review-relay` and
  `codexprompt` skills is intentional: `xcode-review` replaces both with a
  single Author-role handoff skill that emits the Codex prompt inline.

**Known risks / tradeoffs.**
- The TOML merge is deliberately text-based rather than a full `tomllib` →
  `tomli_w` round trip, because preserving comments/order in a user's real
  `~/.codex/config.toml` matters for consent-first install. This means unusual
  inline project-table forms are refused rather than rewritten.
- `init --agent codex` does not auto-install the `notify` fallback or
  `AGENTS.override.md`; both remain documented-only fallbacks per ADR-0005/§5.
- This has not yet been run against the user's real `~/.codex` config; the live
  installed Codex init + in-vivo cross-agent demo still needs the user's
  environment.
- The branch currently uses `git diff --cached` as the review range because the
  implementation was left staged but uncommitted when Claude ran out of credit.

## Reviewer findings  _(Reviewer — Codex)_

1. **major** — `src/neurobase/cli/__init__.py:217`
   `init --agent codex` uses the literal current working directory as the
   project root. Project-scoped Codex hooks are specified as
   `<repo>/.codex/hooks.json` with a `~/.codex/config.toml` project table for
   that repo, but running the installer from `repo/subdir` currently writes
   `repo/subdir/.codex/hooks.json` and configures the subdirectory path. That
   can leave the actual repo session unhooked, and it diverges from the existing
   `enable` path's `git_common_root()` normalization. Suggested direction:
   normalize project-scope installs through `projects.git_common_root(cwd) or
   cwd.resolve()` before computing both the hooks path and `project_key`, and add
   a regression that invokes `init --agent codex --cwd` from a nested git
   directory.

2. **minor** — `src/neurobase/adapters/codex/install.py:189`
   The custom TOML table-header parser terminates a quoted segment at the first
   `"` byte, even when that quote is escaped. The first install for a project
   path containing a quote can append a valid escaped
   `[projects."...\"..."]` table, but a later idempotence pass will fail to find
   that same header, see the project already exists via `tomllib`, and raise
   `ConfigParseError`. Suggested direction: teach `_parse_dotted_key()` to honor
   TOML string escapes, or avoid the custom parser for the rendered project
   header; add a regression that applies `merge_config()` twice to a project key
   containing `"`.

Verification run:
`uv run pytest -q`, `uv run ruff check .`, and `uv run mypy src` all pass.

**Verdict:** changes-requested — project-scoped Codex init can install into the
wrong directory when run below the repo root.

## Author resolution  _(Codex, at user request)_

**F1 — resolved.** Project-scope Codex init now normalizes the install target
through `projects.git_common_root(resolved_cwd) or resolved_cwd` before deriving
both `<repo>/.codex/hooks.json` and the `~/.codex/config.toml` project key. Added
`test_init_codex_project_scope_uses_git_root_from_subdir`, which invokes
`init --agent codex --cwd` from a nested git directory and verifies the hook and
project table land at the repo root, not the subdirectory.

**F2 — resolved.** `_parse_dotted_key()` now handles TOML basic-string escapes
before treating `"` as the string terminator, so escaped quotes inside project
paths are parsed correctly. Added
`test_merge_config_idempotent_with_escaped_quote_path`, which applies
`merge_config()` twice to a quoted path and verifies byte-for-byte idempotence.

Verification run:
`uv run pytest -q tests/test_codex_install.py tests/test_cli_init.py`,
`uv run pytest -q`, `uv run ruff check .`, and `uv run mypy src` all pass.
Scoped `uv run ruff format --check` passes for the touched implementation and
test files. Full `uv run ruff format --check .` still reports pre-existing
formatting differences in unrelated files:
`src/neurobase/adapters/codex/scribe.py`, `tests/test_cli_hook_codex.py`, and
`tests/test_codex_scribe.py`.

**Re-relay:** please re-review the Codex project-root normalization in

## Re-review  _(Reviewer — Codex)_

F1 is resolved. The staged `_init_codex()` path now uses
`projects.git_common_root(resolved_cwd) or resolved_cwd` for project-scope
installs, and feeds that normalized root into both `hooks_json_path()` and the
`config.toml` project key. The regression
`test_init_codex_project_scope_uses_git_root_from_subdir` covers the original
failure mode by invoking init from a nested git directory and asserting the hook
and config entry land at the repo root, not the subdirectory.

F2 is resolved. `_parse_dotted_key()` now checks for a basic-string terminator
inside the loop only after giving escape sequences a chance to decode, so
`\"` inside a project path no longer ends the parsed segment early. The
regression `test_merge_config_idempotent_with_escaped_quote_path` verifies the
two-pass idempotence case that previously raised `ConfigParseError`.

Verification run:
`uv run pytest -q tests/test_codex_install.py tests/test_cli_init.py`,
`uv run pytest -q`, `uv run ruff check .`, `uv run mypy src`, and
`git diff --cached --check` all pass.

**Final verdict:** approve — both requested fixes are present, targeted, and
covered by regressions.
